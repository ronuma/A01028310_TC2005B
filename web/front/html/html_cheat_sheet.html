strong - Énfasis y negritas al texto

Sirve para dar énfasis al texto colocándolo en negritas.

i - Idiomatic text

Es un grupo de texto que se destaca del texto principal, el texto entonces se muestra en Itálicas.

u - Unarticulated Annotation (antes Underline)

Utilizado para subrayar con rojo partes del texto, como para indicar errores y demás. Antes, como sugeríe el nombre, sirve para subrayar el texto. Inline, como prácticamente todos los elementos que estilizan directamente el texto, porque, de otro modo, siendo block, mandaría el texto abajo y perdería su propósito.

b - Antes Boldface

Casi como strong, se utiliza para poner texto en negritas para llamar su atención. Aunque se sugiere que las negritas ya se apliquen con css, y si se le quiere dar importancia especial al contenido, utilizar strong.

strike - Línea sobre el texto

Sirve para tachar parte del texto, estilizándolo mostrándolo de esta forma. Con el propósito que cada quien ocupe busque en sus textos. Ya está deprecado.

small - Small text element

Sirve para colocar el texto fuera de su tamaño para diferenciarlo del resto del contenido de la página, como texto de copyright o aviso de privacidad.

Este programa es público, ajeno a cualquier partido político. Queda prohibido su uso para fines distintos a los establecidos en el programa.

em - Emphasis

Sirve para dar énfasis al texto empleado, colocándolo en cursiva.

header - Sección de título

Sirve para colocar, como bloque, el título de la página o de una sección de la página.

Aunque esto es un h1

Y esto es un párrafo, ambos están dentro de un elemento header, que permite agruparlos y dar a entender a quien lee el HTML que esta sección es la de título.

section - Sección

Al igual que el header, sirve para delimitar una sección de contenido de la página.

Igual esto no es visible, pero este texto no usa un p tag solamente si no que es un p tag dentro de un section tag.

article - Artículo

Permite dar sentido y orden a la página para poner un contenido, usualmente en forma de artículo o blog post.

aside - Aside

Sirve para colocar una parte del documento más o menos relacionada con el contenido principal, o bien, colocarlo a un lado.

Como este contenido que está dentro de un aside tag.

footer - Footer

Podríamos decir, el opuesto del header. Es semántico y sirve para envolver contenido que va a ir hasta el pie de la página, y su uso es más semántico que nada.

Todos los derechos reservados. Este es el footer de la página, que contiene un small tag dentro del footer tag.

nav - Navigation Section

Más bien representa una sección de la página en donde hay navegación principal, ya sea para navegar dentro de la página o salir de la página.

hr - Horizontal Rule

Aquí vimos su uso y su ejemplo, dividir contenido en la página.

form - Formulario

Este es un tag que permite incluir varios inputs en donde se ingresen datos, y se pueda controlar lo que sucede con las interacciones del usuarios, como cuando dé click a un botón de Submit o de enter.

table - Tabla

Tal cual es una tabla, en la que podemos representar la información en celdas que están organizadas como filas y columnas. Se puede usar thead para incluir un header a la tabla, tr para cada fila de la tabla, y td para cada celda de la tabla. Hay más tags que son semánticas para entender cómo está organizada la tabla. Ver el código fuente de esta tabla para verlas todas y dedudcir sus significados.
